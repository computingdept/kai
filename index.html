Recursion, how it can be used and compares to an iterative approach. explain simply typocal question Thought for a couple of seconds Recursion — simple explanation Recursion is when a subroutine calls itself to solve a problem. It keeps breaking the problem into smaller versions of the same problem until it reaches a stopping point. That stopping point is called the base case. Simple analogy Like Russian dolls: You keep opening a smaller doll inside, until you reach the smallest one. That smallest doll is the base case. Example: countdown Recursive approach def countdown(n): if n == 0: print("Done") else: print(n) countdown(n - 1) This calls itself again and again: countdown(3) countdown(2) countdown(1) countdown(0) Done Iterative approach Iteration means using a loop, such as FOR, WHILE or REPEAT. def countdown(n): while n > 0: print(n) n = n - 1 print("Done") This repeats using a loop instead of calling itself. Recursion vs iteration Recursion Iteration A subroutine calls itself A loop repeats code Needs a base case Needs a loop condition Can be elegant for some problems Usually simpler and more efficient Uses more memory because calls are stored on the stack Usually uses less memory Risk of stack overflow if it never stops Risk of infinite loop if condition is wrong Typical exam question Question: Explain how recursion works and compare it with an iterative approach.
